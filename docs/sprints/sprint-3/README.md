# 📅 Documentação - Sprint 3

> Status da Sprint: Em andamento 🟡

## 🏅 Desafio

Evoluir a aplicação Radar Cidadão com correções e melhorias voltadas à visualização e análise de dados parlamentares.

Nesta sprint, o foco esteve na implementação de funcionalidades comparativas e no aprimoramento da experiência do usuário, incluindo médias de indicadores da Câmara, ranking de deputados e melhorias na organização das informações exibidas. Também foram planejadas funcionalidades de personalização e compartilhamento, tornando a plataforma mais útil e acessível para acompanhamento da atuação parlamentar.

## 🏅 DoR - Definition of Ready

| Critério | Descrição |
|---------|----------|
| Clareza na Descrição | A User Story descreve claramente a funcionalidade relacionada aos dados dos deputados. |
| Critérios de Aceitação Definidos | Estão definidos critérios como exibição correta de dados, funcionamento da comparação e carregamento das informações. |
| Cenários de Teste Especificados | Existem cenários de teste para validar funcionalidades como comparação, exibição de projetos e indicadores. |
| Independente | A funcionalidade pode ser desenvolvida sem depender de outra da Sprint. |
| Compreensão Compartilhada | A equipe entende como os dados devem ser exibidos e comparados no sistema. |
| Estimável | A história possui estimativa baseada na complexidade da funcionalidade. |
| Documentos de Apoio | Protótipos (Figma) ou referências das telas estão disponíveis. |

---

## 🏅 DoD - Definition of Done

| Critério | Descrição |
|---------|----------|
| Critérios de Aceitação atendidos | A funcionalidade atende aos critérios definidos, como exibição correta dos dados e funcionamento da comparação. |
| Testes manuais | Funcionalidades como comparação, visualização de projetos e indicadores foram testadas manualmente. |
| Código revisado | O código foi revisado pela equipe. |
| Documentação interna atualizada | Atualizações, estrutura de dados ou funcionalidades foram documentadas e atualizadas no Jira. |
| Integração com outras partes | A comunicação entre frontend e backend foi validada e a aplicação continua funcionando corretamente após a implementação. |
| Validação do PO | O Product Owner valida se os dados apresentados atendem ao objetivo do sistema. |
